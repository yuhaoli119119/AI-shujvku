# D3-4.2 Frontend Toolchain Reproducibility Note / Verification Command Lock

## 1. 当前问题

- `npm` 在部分终端/Agent shell 环境中不可索引（不在 PATH 中）。
- 标准命令 `npm test -- --project=chromium` 无法直接运行，往往导致测试流程阻塞或反复报错。
- 这是一个**本地工具链 PATH 问题**，不代表 D3 safety contract 或前端业务代码存在任何功能性缺陷。

## 2. 已验证的等价路径

为了避免后续验证或机器审查反复卡在 `npm` 环境依赖上，D3-4.1.1 已验证以下使用 bundled Node / Playwright 的等价执行路径：

### 步骤 A：启动 Frontend 静态 Server
在能够使用 `python` 或 `py` 的环境中，启动简单的 HTTP Server 承载静态文件：
```bash
cd literature-ai/frontend
py -m http.server 8000
```
*(如果环境内也没有 `py`，则可使用任何轻量静态文件服务器代理 `frontend` 目录并在本地 8000 端口提供服务)*

### 步骤 B：使用 Bundled Node / Playwright 执行测试
在拥有 `node.exe` 独立执行环境（或特定解压目录）的终端中，通过绝对路径或相对路径直接调用 Node 和 Playwright CLI 核心脚本。

**通用命令范例（假设当前在 `frontend` 目录下）：**
```bash
# 替换 <node_path> 为实际 node.exe 的绝对路径
# 替换 <playwright_cli_path> 为实际 node_modules/@playwright/test/cli.js 的路径
<node_path>\node.exe <playwright_cli_path> test --project=chromium
```

**示例（如果依赖安装在外部共享目录）：**
```bash
cd literature-ai/frontend
C:\path\to\node.exe C:\path\to\node_modules\@playwright\test\cli.js test --project=chromium
```

## 4. 适用场景

- **场景 1：当 `npm` 可用时（优先）**
  直接按照标准流程验证：
  ```bash
  cd literature-ai/frontend
  npm test -- --project=chromium
  ```
- **场景 2：当 `npm` 不在 PATH 时（Fallback）**
  遇到 "npm: command not found" 等环境受限错误时，切勿反复尝试安装，而应**跳过本地测试步骤**（记录测试未执行的原因），或使用上述 **bundled Node / Playwright fallback 命令**（如果用户提供了对应的环境路径）。
