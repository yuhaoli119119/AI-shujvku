## 3. UX Behavior

Human Workbench now shows a dedicated `Prepare review queue` panel near the current paper/review queue area, before audit and warning details.

Button and helper copy:

- Button: `Prepare pending human reviews`
- Helper: creates pending, unverified review rows from extraction output.
- Helper explicitly says it does not verify, export, unlock writing, reprocess, materialize, or run migrations.

Behavior:

- Page load does not call `POST /api/extraction/results/{paper_id}/reviews/prepare`.
- Clicking the button opens a browser confirm first.
- Canceling confirm does not call prepare and does not show a success summary.
- Confirming calls prepare once with no request body.
- During prepare the button is disabled and shows `Preparing...`.
- Success displays the D4-3D.1 idempotent summary fields:
  - `Created pending rows`
  - `Existing pending rows reused`
  - `Skipped`
  - `Verified rows created`
  - `Safe verified rows`
  - `Review items returned`
- If pending rows already exist, the button is disabled and shows `Pending queue already prepared`.

## 4. Safety Assertions

Frontend prepare:

- does not send `reviewer_status=verified`
- does not send `verified=true`
- does not send `safe_verified=true`
- does not send `mark_verified`
- does not send export, writing, materialize, reprocess, or migration fields
- does not unlock export
- does not unlock writing
- does not run extraction/reprocessing
- does not run materialize
- does not run migrations

Pending review queue remains visibly unsafe until real human verification and exact locator requirements are satisfied:

- pending rows still show `Pending human review / Not verified`
- missing locator warning still shows `Exact PDF locator missing / missing_page / unsafe_locator / no exact locator`
- PDF jump/highlight remains absent for missing exact locator rows
- export/writing blocked wording remains visible
- no prepare wording claims Human verified, export-ready, writing-ready, final export, or final conclusion

If a prepare response unexpectedly reports nonzero `verified_count` or `safe_verified_count`, the panel displays an unsafe-state warning.
