## Coverage Inventory

### A. Empty Export State Contract

- When `exported = 0` and `blocked = 0`, the UI accurately reflects the empty state.
- Status displays `Safety gate: enforced | exported: 0 | blocked: 0 (没有可导出的 Human verified + required evidence 记录)`.
- The export toast clearly specifies `0 rows exported / 没有记录被导出`.
- Playwright coverage ensures the absence of any generic "Export successful" or "Direct export" wording.
- D3-4.1.1 verified the smoke scenario `business flow: DFT export empty state shows correct wording and status` runs and passes.

### B. Blocked-Only Export State Contract

- When `exported = 0` and `blocked > 0`, the UI avoids implying that blocked rows were exported.
- Status displays `Safety gate: enforced | exported: 0 | blocked: {count} (没有 safe rows 被导出，因缺少 Human verified 或 required evidence)`.
- The export toast clearly specifies `0 rows exported / 没有 safe rows 被导出 ({count} blocked)`.
- Playwright coverage locks the UX scenario to prevent misleading success language and confirms the reason explicitly references the safety requirements (Human verified / required evidence).
- D3-4.1.1 verified the smoke scenario `business flow: DFT export blocked-only state shows correct wording and status` runs and passes.

### C. Safe Gate Header Surface

- The DFT export UI reads and displays `X-D3-Export-Safety-Gate`.
- The DFT export UI reads and displays `X-D3-Export-Count`.
- The DFT export UI reads and displays `X-D3-Block-Count`.
- D3-4.1.1 verified the empty state uses `safe gate = enforced`, `exported = 0`, `blocked = 0`.
- D3-4.1.1 verified the blocked-only state uses `safe gate = enforced`, `exported = 0`, `blocked = 3`.

### D. Misleading Wording / Bypass Guard

- The DFT export scenarios assert no generic success wording such as `Export successful`, `导出成功`, `已导出`, or `全部完成` appears in the scoped UI after empty or blocked-only export.
- The DFT export scenarios assert no bypass entry wording such as `Direct export`, `Export final`, or `Final conclusion` appears in the scoped UI.
