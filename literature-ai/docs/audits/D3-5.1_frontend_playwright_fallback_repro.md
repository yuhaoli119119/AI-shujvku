## 3. 问题说明
- **npm 不在 PATH**：在此 Agent / Shell 运行环境中，`npm` 无法被直接调用。
- **bundled Playwright fallback 可运行**：通过直接调用 `node.exe` 以及 Playwright CLI，可以在没有 `npm` 的情况下运行前端测试。
- **临时依赖问题**：但如 D3-5 中所记录，直接运行 fallback 命令需要指定由于 pnpm layout 导致的 `NODE_PATH`，并且还需要一个临时、非仓库的 `@playwright/test` shim 目录才能让测试脚本成功 `require('@playwright/test')` 并绕过 webServer 的 `npm run test:serve` 启动检查。
- 该问题纯粹属于本地工具链可复现性（Reproducibility）问题，不代表 D3 safety contract 或任何业务逻辑存在缺陷。

## 4. 本轮处理
- **新增脚本**：新增了轻量本地验证脚本 `literature-ai/scripts/run_frontend_chromium_fallback.ps1`。
- **脚本行为**：
  - 检查环境中是否有 `npm`，如果有，则提示并退出，建议使用标准命令。
  - 如果没有 `npm`，则自动生成一个存放在 `%TEMP%` 目录下的临时 shim 文件夹。
  - 在此临时文件夹中创建 `@playwright/test` 的包重定向机制。
  - 在此临时文件夹中创建假的 `npm.cmd`，从而在 Playwright 尝试执行 webServer 阶段启动 `py -m http.server 8000` 作为后台服务，避免 webServer 启动失败阻断测试流程。
  - 将生成的临时 shim 加入 `PATH` 与 `NODE_PATH`，调用 Bundled Node 和 Playwright CLI 运行 `--project=chromium`。
  - 运行结束后自动清理、删除临时目录。
- **关于临时 shim**：仍然在技术机制上依赖了 shim，但此时由脚本在系统临时目录中动态生成并销毁，避免了手动干预。
- **Shim 仓库污染**：没有任何 shim 被提交进代码仓库。业务代码保持干净，无任何额外 npm dependency。
