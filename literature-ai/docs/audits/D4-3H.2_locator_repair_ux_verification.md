## 2. Workbench UX Implementation Overview

### Asynchronous Locator Hydration
- Awaited `loadExtractionLocators()` asynchronously inside `loadSelectedPaper()` and `refreshReviewStateAfterMutation()` before `renderSchemaForm()` starts.
- This ensures the UI resolves evidence locators prior to execution, preventing display delays.

### Multiple Key Cache Mapping
- Hydrated `state.locatorCache` using multiple key variations:
  - `loc.target_id + ":" + loc.field_name` (precise UUID match)
  - `loc.target_type + ":" + loc.field_name` (canonical schema match)
  - `loc.target_type.toLowerCase() + ":" + loc.field_name` (lowercase fallback match)

### Repaired Locator Rows Display
The 3 repaired rows (`catalyst_type`, `metal_centers`, `rate`) display:
- **Wording**: `locator repaired / exact locator available / page available` (clear but not exaggerated)
- **Visuals**: A soft green badge with standard premium styling.
- **Details Info Box**: Renders below the field title, displaying:
  - `📄 Page Span: page 7` (or `page 6`)

  - `Matched text: "evidence text snippet"`
- **Interaction**: Renders a `Locator Inspection (Jump to Page X)` button. This ensures jump entries are clearly labeled as "inspection" rather than "verification".
