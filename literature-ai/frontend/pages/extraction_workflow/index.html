按需原子工具决策树

AI Agent 在提取流程中可随时调用以下工具查证。下方为简化决策指引，完整行为规范见 Agent Behavior Guide。

🔍 宏观检索

用 retrieve_evidence(query) 跨文献语义检索证据切片。适合定位"哪些论文提到 X"。

🌐 外部文献发现

用 search_external_papers(query, year_min, year_max, target_types) 从 OpenAlex、arXiv 等外部数据库检索新文献。支持年份过滤和类型分类（computational/experimental/review）。只检索不入库，后续用 parse_paper 或 ingest_pdf_batch 导入。

📖 翻页查证

当 RAG 片段被截断或缺上下文时，用 read_paper_page(paper_id, page) 翻开原文指定页。返回该页所有 sections、tables 和 figures 的摘要。

🔬 图表追问

当 AI Visual Summary 不够具体时，用 analyze_chart(figure_id, question) 唤醒 VLM 重新凝视原图。如"绿色曲线与 x 轴的交点值"。结果自动落盘为共享笔记，其他 AI 和人类可通过 list_notes 查看分析痕迹——雁过留声。

✅ 图表审核

用 review_figure(figure_id, verdict, reasoning) 对图表记录审核结论（verified / needs_attention / incorrect）。结论自动落盘为共享笔记，其他 AI 可见。

✂️ 图表重裁

用 recrop_figure(figure_id, strategy, new_bbox) 修正错误的图片裁剪。支持整页、扩大、缩小。AI 可先在本地试裁，确认无误后使用 ai_bbox 策略注入系统闭环。

📋 审核覆盖率

用 get_review_coverage(paper_id) 查看哪些图表、表格、章节已被审核、哪些还没人碰。快速定位盲区。

⚔️ 分歧检测

用 get_field_disputes(paper_id) 找出多个 AI 对同一字段给出了不同值的冲突，包括已裁决的历史分歧（status='resolved'）。后续 AI 可避免重复踩坑。

✅ 批量确认

用 verify_dft_results_batch / reject_dft_results_batch 批量审核 DFT 结果；用 approve_corrections_batch / reject_corrections_batch 批量处理修正提案。减少重复操作。

📊 ML 数据集导出

用 export_ml_dataset(target_types=['dft_results'], format='json') 导出已审核通过的 ML 就绪数据。支持按年份、库名过滤，输出 JSON 或 CSV。
