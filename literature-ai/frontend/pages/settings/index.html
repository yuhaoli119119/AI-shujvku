使用说明

系统怎么配合使用

这套系统把工作流拆成入库、解析、审阅、确认和写作五个阶段。先把论文放进文献库并完成解析，再看结构化结果，再让 AI 生成候选建议，最后经人工确认后进入正式数据或写作流程。

主要功能分区

  • 文献库：上传 PDF、按 DOI / URL 入库、阅读摘要、章节和图表。
  • DFT 数据库 / 机理知识：适合做横向比对和聚合浏览。
  • AI 审阅与确认：生成候选建议，再走人工核对。
  • AI Writer：基于文献和证据链生成草稿。
  • 设置：集中管理 API、MCP 和界面主题。

启动方式

docker compose up -d --build
cd backend python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

常用入口

  • /api/health
  • /pages/literature_library/index.html
  • /pages/paper_detail/index.html?id=<paper_id>
  • /pages/ai_writer/index.html
  • /pages/settings/index.html

文献入库顺序

  1. 先确认当前使用的是目标文献库。
  2. 用上传 PDF、DOI / URL 下载或在线搜索把论文放进库里。
  3. 等待解析完成后检查摘要、章节、图表和结构化字段。
  4. 需要横向比较时，再切换到聚合页面。

AI 审阅与确认流程

  1. 在文献库选中论文。
  2. 生成网页内 AI 审阅或导入外部 AI 结果。
  3. 筛选候选内容，生成待确认记录。
  4. 最后在人工确认工作台核对证据并确认。

写作与证据

AI Writer 依赖已入库文献、检索结果和证据链。还在 review / pending 状态的数据，不应直接当作正式结论使用。

IDE / MCP 连接

  1. 先配置 MCP API Keys。
  2. 复制 IDE 连接页的提示词或配置。
  3. 交给 Cursor、VS Code、Claude Desktop 等客户端。
  4. 最终确认仍建议回到网页工作台完成。

排查问题

  1. 先看 docker ps。
  2. 再访问 /api/health。
  3. 查看 docker compose logs backend --tail=200。
  4. 如果改了代码但行为没变，优先重启 backend。
