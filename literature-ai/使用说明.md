# Literature AI 使用说明

本文面向日常使用者，描述当前可用入口、启动方式和常见问题。以当前运行事实为准：默认活跃业务库是 SQLite，不是 PostgreSQL。

## 1. 启动服务

在项目目录执行：

```powershell
cd literature-ai
docker compose up --build
```

常见容器：

- `backend`
- `worker`
- `redis`
- `minio`
- `grobid`
- `postgres`（可启动，但不是默认活跃业务库）

## 2. 健康检查

启动后访问：

- <http://localhost:8000/api/health>

如果返回健康状态，说明后端已经可用。

## 3. 常用入口

- 文献库工作台：<http://localhost:8000/pages/literature_library/index.html>
- 论文详情页：`http://localhost:8000/pages/paper_detail/index.html?id=<paper_id>`
- 外部 AI 工作台：<http://localhost:8000/pages/external_analysis_workbench/index.html>
- AI Writer：<http://localhost:8000/pages/ai_writer/index.html>

## 4. 文献库怎么用

在文献库工作台中，你可以：

- 上传本地 PDF
- 按 DOI 下载并入库
- 做在线文献搜索
- 查看论文详情、Sections、Tables、Figures
- 查看 DFT、机理、电化学、Writing Cards 等抽取结果

当前活跃文献库的数据会写入对应 library 目录下的 `database.sqlite`。

## 5. 论文详情怎么看

论文详情页或主工作台详情面板通常可查看：

- 论文基础元数据
- 原文分段与结构
- 抽取出的结构化字段
- 部分证据定位与 PDF 预览能力

如果 PDF 缺失，系统会提示无法预览，而不是自动补全不存在的文件。

## 6. 外部 AI 工作台怎么用

入口：

- <http://localhost:8000/pages/external_analysis_workbench/index.html>

典型流程：

1. 选择一篇论文
2. 生成提示词或导出审阅上下文
3. 把提示词和论文内容交给外部 AI
4. 将外部 AI 返回的文本或 JSON 粘回工作台
5. 导入候选结果
6. 审核候选项后再决定是否物化

注意：AI 生成内容不等于人工 verified。

## 7. Docker 相关常见问题

如果页面打不开，先检查容器是否启动：

```powershell
docker compose ps
```

如果后端未正常启动，查看日志：

```powershell
docker compose logs backend --tail=200
```

## 8. 常见问题

### 为什么我看见 PostgreSQL 容器，但系统实际还在用 SQLite？

因为当前默认活跃业务库仍是每个 library 目录下的 SQLite。PostgreSQL 目前是可选能力，不是默认主库。

### 能不能直接做 migration apply？

当前阶段不建议直接 apply。先做 readiness、审计、文档核对和测试，再决定是否执行。

### 能不能移动 active SQLite 或手动清理 artifacts？

默认不要。除非任务明确要求，并且先确认影响范围。
