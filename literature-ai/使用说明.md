# Literature AI 使用说明

## 1. 这套软件现在能做什么

- 本地文献库管理
- 在线检索文献
- DOI 下载并自动入库
- PDF 解析
- DFT / 电化学 / 机理 / 写作卡片抽取
- 外部 AI 审稿工作台
- MCP 对接 IDE 内 AI

---

## 2. 启动方式

在目录：

`D:\Desktop\03_代码与开发\爬虫脚步下载文章\literature-ai`

运行：

```powershell
docker compose up -d --build
```

查看服务状态：

```powershell
docker compose ps
```

如果正常，应该至少看到这些服务：

- `backend`
- `worker`
- `postgres`
- `redis`
- `minio`
- `grobid`

## 4. 如果网页打不开，先看什么

最常见原因不是页面坏了，而是服务没启动。

先运行：

```powershell
docker ps
```

如果输出为空，说明容器都没在运行。

这时重新启动：

```powershell
docker compose up -d
```

再访问：

- [http://localhost:8000/api/health](http://localhost:8000/api/health)

如果健康检查都打不开，就不要先怀疑前端，先看后端是否启动成功。

查看后端日志：

```powershell
docker compose logs backend --tail=200
```

## 5. 日常使用流程

### 5.1 在线搜索文献

打开文献库页面：

- [http://localhost:8000/pages/literature_library/index.html](http://localhost:8000/pages/literature_library/index.html)

可以：

- 搜标题/关键词
- DOI 下载
- 上传 PDF
- 本地文献浏览

### 5.2 下载并入库 DOI

示例接口：

```powershell
curl -X POST http://localhost:8000/api/papers/discovery/download `
  -H "Content-Type: application/json" `
  -d "{\"identifier\":\"10.1038/s41467-025-65194-0\"}"
```

### 5.3 看解析结果

在文献库中点开论文详情即可查看：

- Sections
- Tables
- Figures
- DFT Settings
- DFT Results
- Electrochemical Performance
- Mechanism Claims
- Writing Cards

## 6. 外部 AI 审稿工作台怎么用

入口：

- [http://localhost:8000/pages/external_analysis_workbench/index.html](http://localhost:8000/pages/external_analysis_workbench/index.html)

使用步骤：

1. 选择一篇论文
2. 点击“生成提示词”或“一键生成并复制提示词”
3. 把 PDF + 提示词发给外部 AI
4. 把外部 AI 返回的 JSON 或文本粘回工作台
5. 点击 `Import Result`
6. 审查候选项
7. 点击 `Materialize Pending` 或 `Materialize Selected`

现在工作台支持直接显示：

- note 摘要
- correction 原值 / 新值对比
- evidence 证据块
- relationship 目标信息

## 7. IDE 里的 AI 怎么连接

### 7.1 前提

你的本地服务必须先启动：

- `http://localhost:8000/mcp`

并且 `.env` 中要有 MCP key，例如：

```env
LITAI_MCP_ENABLED=true
LITAI_MCP_SERVER_NAME=Literature AI MCP
LITAI_MCP_API_KEYS=claude|Claude Desktop|litmcp_claude|read_papers,append_notes,propose_corrections,request_parse;admin|Admin|litmcp_admin|read_papers,append_notes,propose_corrections,request_parse,review_corrections
```

---

### 7.2 CodeBuddy 配置

文件位置：

`C:\Users\28128\.codebuddy\mcp.json`

建议内容：

```json
{
  "mcpServers": {
    "literature-ai": {
      "command": "npx",
      "args": [
        "-y",
        "mcp-remote",
        "http://localhost:8000/mcp",
        "--transport",
        "http-only",
        "--allow-http",
        "--header",
        "Authorization:${LITAI_AUTH_HEADER}"
      ],
      "env": {
        "LITAI_AUTH_HEADER": "Bearer litmcp_claude"
      },
      "disabled": false
    }
  }
}
```

---

}

## 8. MCP 连上后 AI 能做什么

普通协作者 key 可用：

- `query_papers`
- `get_paper`
- `list_notes`
- `append_note`
- `propose_correction`
- `scan_local_pdfs`
- `ingest_pdf_batch`

审核者 key 额外可用：

- `get_correction_queue`
- `get_correction_detail`
- `approve_correction`
- `reject_correction`

## 9. 局域网其他电脑怎么访问

如果你要让同一 WiFi 下其他电脑访问，把：

```text
http://localhost:8000/mcp
```

改成你电脑的局域网 IP，例如：

```text
http://192.168.1.23:8000/mcp
```

同时注意：

- Windows 防火墙要放行 `8000`
- 你的电脑和对方电脑在同一局域网
- `docker compose` 服务必须正在运行

## 11. 当前最推荐的使用方式

最稳的工作流是：

1. 本地系统负责存储与解析
2. IDE 里的 AI 通过 MCP 读写文献与审稿候选项
3. 外部网页 AI 通过工作台参与复核
4. 最终由你或审核 AI 决定哪些修改进入正式数据

这样可以避免：

- 单一 AI 幻觉
- 外部 AI 直接污染数据库
- 支撑文献关系丢失
